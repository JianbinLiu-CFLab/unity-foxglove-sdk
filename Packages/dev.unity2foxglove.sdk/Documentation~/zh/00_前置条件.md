# 0. 

## 阅读对象

在首次安装 Unity2Foxglove、打开演示项目或构建 IL2CPP Player 之前，请阅读本文。

## 你将完成

你将检查每种工作流所需的 Unity 版本、Foxglove 应用、IDE、命令行工具以及可选模块。

## 0.1 所有人必备

| 工具             | 推荐版本                                                         | 为什么需要                                                    | 备注                                                              |
| ---------------- | ------------------------------------------------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------ |
| Unity Editor     | Unity 6000.0 LTSC 或更高版本（开发环境为 6000.3.14f1 LTSC；兼容 6000.0.74f1 LTSC） | 打开项目、导入包并运行 Play Mode。   | 不支持 Unity 2022。 |
| Foxglove Desktop | 最新稳定桌面版                                           | 通过 Foxglove WebSocket 连接 Unity 并显示面板。 | 使用 `ws://127.0.0.1:8765` 作为默认本地连接。        |
| Git              | 任意较新版本                                                  | 克隆仓库或跟踪包变更。               | 如果从仓库路径安装，则需要此项。                      |

使用 Unity2Foxglove 不需要 ROS。

## 0.2 Unity 模块

对于 Editor Play Mode，标准 Unity 安装即可满足需求。

对于 Player 构建，请在 Unity Hub 中安装目标平台模块：

| 目标平台 | Unity Hub 模块 |
|---|---|
| Windows | Windows Build Support with IL2CPP |
| Linux | Linux Build Support with IL2CPP |
| macOS | macOS Build Support |

如果构建脚本在编译 C# 之前失败，请首先检查目标平台模块是否已安装。

## 0.3 IDE 与 C# 编辑

IDE 对于运行示例是可选的，但如果你想编写脚本，建议安装。

推荐选项：

- Visual Studio 并安装 Unity 工作负载
- JetBrains Rider
- Visual Studio Code 并安装 C# 和 Unity 扩展

IDE 的用途：

- 编写自定义 publisher 脚本
- 添加 `[FoxRun]` 调试字段
- 注册 Parameters 和 Services
- 检查编译错误

## 0.4 命令行工具

| 工具 | 用于 | 检查方法 |
|---|---|---|
| Python 3 | `Scripts/build_unity_il2cpp.py` | `python --version` |
| .NET SDK | 运行时验证测试 | `dotnet --version` |
| PowerShell 或终端 | 构建和测试命令 | 在仓库根目录打开 shell |

仅当你使用仓库构建脚本时才需要 Python。仅当你运行包测试项目时才需要 .NET SDK。

## 0.5 包依赖

Unity 会自动解析主要包依赖：

- `com.unity.nuget.newtonsoft-json`

该包还包含用于 MCAP 压缩支持的运行时插件程序集。如果你只使用实时 Foxglove 流传输，可以暂不考虑压缩相关问题。

## 0.6 可选演示依赖

Full Demo 示例和 `Unity2Foxglove` 演示项目比 Basic 示例使用了更多的 Unity 特性。

| 特性 | 使用者 | 备注 |
|---|---|---|
| Input System | 鼠标拖拽演示控制 | 完整的交互式演示需要此项。 |
| URP | 完整演示视觉效果 | Basic 示例故意保持精简。 |
| MCAP 文件 | 录制/回放演示 | 在需要离线回放之前可以跳过此项。 |

如果你希望进行最简单的初次测试，请使用 [03 基础可视化](03_基础可视化.md)。

## 0.7 网络假设

默认服务器地址为：

```text
ws://127.0.0.1:8765
```

这表示 Foxglove 和 Unity 运行在同一台机器上。

如需从另一台机器连接：

1. 将 `FoxgloveManager > Host` 设置为 `0.0.0.0`。
2. 保持或修改端口号。
3. 在防火墙中允许该端口。
4. 将 Foxglove 连接到 `ws://<unity-machine-ip>:8765`。

请仅在受信任的网络上使用此配置。

## 0.8 快速就绪检查清单

- Unity 打开项目时无包错误。
- Foxglove Desktop 已安装。
- 可以在 Unity 中按下 Play。
- 清楚自己使用的是 Basic 示例、Full Demo 示例还是 `Unity2Foxglove`。
- 如果要构建 IL2CPP，Python 和目标 Unity 构建模块已安装。

通过此检查清单后，请继续阅读 [01 安装与快速开始](01_安装与快速上手.md)。
