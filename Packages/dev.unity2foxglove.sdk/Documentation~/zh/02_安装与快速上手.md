## 1. 目的

本页用于第一次安装 package，并从 Unity scene 建立第一条实时 Foxglove 连接。

## 2. 流程

你将安装 package，添加 `FoxgloveManager`，发布一个 Transform，可选发布 Scene cube 和 Camera image，然后连接 Foxglove Desktop。

> [!TIP]
> 如果还没有安装 Unity 或 Foxglove Desktop，请先阅读 [01_前置条件](01_前置条件.md)。

## 3. 安装 Package

### 3.1 通过 Git URL 安装

1. 打开 Unity。
2. 打开 **Window > Package Manager**。

![open-package-manager](../Pictures/01/open-package-manager.png)
<figcaption>图 1：打开 Package Manager</figcaption>

3. 点击 **+ > Install package from git URL...**，输入：

```http
https://github.com/JianbinLiu-CFLab/unity-foxglove-sdk.git?path=/Packages/dev.unity2foxglove.sdk
```

![package-manager-add-menu](../Pictures/01/package-manager-add-menu.png)
<figcaption>图 2：Package Manager 添加菜单</figcaption>

4. 等待 Unity 解析依赖。

![package-imported-confirmation](../Pictures/01/package-imported-confirmation.png)
<figcaption>图 3：Package 导入确认</figcaption>

### 3.2 从本地 checkout 安装

如果你已经 clone 了仓库，使用这个方式。

1. 打开 Unity。
2. 打开 **Window > Package Manager**。
3. 选择 **+ > Add package from disk...**。
4. 选择 `Packages/dev.unity2foxglove.sdk/package.json`。

![select-package-json](../Pictures/01/select-package-json.png)
<figcaption>图 4：选择 package.json 文件</figcaption>

5. 等待 Unity 解析依赖。

![package-imported-confirmation](../Pictures/01/package-imported-confirmation.png)
<figcaption>图 5：Package 导入确认</figcaption>

### 3.3 备选方式：编辑 `manifest.json`

通常不推荐这个方式。Unity 可能会把相对 `file:` 路径重写成绝对路径，导致项目不可移植。除非你有明确原因，否则优先使用 Package Manager UI。

```json
{
  "dependencies": {
    "dev.unity2foxglove.sdk": "file:../../Packages/dev.unity2foxglove.sdk"
  }
}
```

根据你的 Unity project 和 package 的相对位置调整路径。

## 4. 添加 Server Component

1. 创建一个空 GameObject，命名为 `Foxglove`。
2. 添加 **FoxgloveManager**。

![add-foxglove-manager](../Pictures/01/add-foxglove-manager.png)
<figcaption>图 6：添加 Foxglove Manager Component</figcaption>

3. 第一次测试保留默认设置：
   - Host: `127.0.0.1`
   - Port: `8765`
   - Start On Enable: enabled
   - Coordinate Mode: `LeftHand`

![foxglove-manager-default-settings](../Pictures/01/foxglove-manager-default-settings.png)
<figcaption>图 7：Foxglove Manager 默认设置</figcaption>

按 **Play**。Unity Console 应显示 server 已在 `ws://127.0.0.1:8765` 启动。

![websocket-server-started](../Pictures/01/websocket-server-started.png)
<figcaption>图 8：WebSocket Server 启动确认</figcaption>

## 5. 发布 Transform

1. 创建一个 Cube。
2. 给 Cube 添加 **Foxglove Transform Publisher**。

![add-transform-publisher](../Pictures/01/add-transform-publisher.png)
<figcaption>图 9：添加 Transform Publisher Component</figcaption>

3. 第一次测试使用这些值：
   - Topic: 留空或使用 `/tf`
   - Parent Frame Id: `unity_world`
   - Child Frame Id: 留空以使用 GameObject 名称
   - Publish Rate Hz: `10`

![transform-publisher-tf-config](../Pictures/01/transform-publisher-tf-config.png)
<figcaption>图 10：Transform Publisher `/tf` 配置</figcaption>

在 Play Mode 中移动或旋转 Cube。

## 6. 连接 Foxglove Desktop

1. 打开 Foxglove Desktop。
2. 点击 **Open connection**。
3. 选择 **Foxglove WebSocket**。
4. 输入 `ws://127.0.0.1:8765`。
5. 点击 **Open**。

确认 Topics panel 中出现 `/tf`。如果要查看原始 transform message，添加 **Raw Messages** panel 并选择 `/tf`。如果要显示 frame，添加 **3D** panel。

![foxglove-tf-topic-connected](../Pictures/01/foxglove-tf-topic-connected.png)
<figcaption>图 11：Foxglove 中连接到 `/tf` topic</figcaption>

核心预期 topic：

- `/tf`，schema 为 `foxglove.FrameTransform`

![tf-frametransform-topic](../Pictures/01/tf-frametransform-topic.png)
<figcaption>图 12：验证 `/tf` FrameTransform topic</figcaption>

在 Play Mode 中用 Move Tool 或 Rotate Tool 移动/旋转 Cube。Transform 变化会立即发布到 `/tf`。

![cube-transform-update-foxglove](../Pictures/01/cube-transform-update-foxglove.png)
<figcaption>图 13：Cube Transform 实时更新</figcaption>

## 7. 可选：发布 Scene Cube

如果你希望 Foxglove 的 3D panel 显示一个简单 cube primitive，而不仅仅是 frame transform，可以给 Cube 添加 **Foxglove Scene Cube Publisher**。

![add-scene-cube-publisher](../Pictures/01/add-scene-cube-publisher.png)
<figcaption>图 14：添加 Scene Cube Publisher Component</figcaption>

推荐第一次测试使用：

- Topic: `/scene`
- Frame Id: 留空以复用 object/frame 名称
- Color: green
- Size: `(1, 1, 1)`

![scene-cube-publisher-config](../Pictures/01/scene-cube-publisher-config.png)
<figcaption>图 15：Scene Cube Publisher `/scene` 配置</figcaption>

回到 Foxglove，添加一个新 panel。

![foxglove-add-panel](../Pictures/01/foxglove-add-panel.png)
<figcaption>图 16：添加 Foxglove panel</figcaption>

选择 3D panel。

![select-3d-panel](../Pictures/01/select-3d-panel.png)
<figcaption>图 17：选择 3D panel</figcaption>

添加 Scene Cube publisher 后，可选预期 topic：

- `/scene`，schema 为 `foxglove.SceneUpdate`

如果 3D panel 中看不到 cube，在左侧 Topics 列表中找到 `/scene`，点击可见性图标启用它。

![enable-scene-topic-3d](../Pictures/01/enable-scene-topic-3d.png)
<figcaption>图 18：在 3D panel 中启用 `/scene` topic</figcaption>

## 8. 可选：发布 Camera Image

1. 选择一个 Unity Camera。
2. 添加 **Foxglove Camera Publisher**。

![add-camera-publisher](../Pictures/01/add-camera-publisher.png)
<figcaption>图 19：添加 Camera Publisher Component</figcaption>

3. 使用：
   - Topic: `/unity/camera`
   - Frame Id: `unity_camera`
   - Publish Rate Hz: `10`
   - Width: `640`
   - Height: `480`
   - JPEG Quality: `70`

![camera-publisher-config](../Pictures/01/camera-publisher-config.png)
<figcaption>图 20：Camera Publisher `/unity/camera` 配置</figcaption>

回到 Foxglove，添加一个新 panel。

![foxglove-add-camera-panel](../Pictures/01/foxglove-add-camera-panel.png)
<figcaption>图 21：添加 Camera 可视化 panel</figcaption>

选择 Image panel。

![select-image-panel](../Pictures/01/select-image-panel.png)
<figcaption>图 22：选择 Image panel</figcaption>

此时预期 topics：

- Core: `/tf`，schema 为 `foxglove.FrameTransform`
- Optional scene publisher: `/scene`，schema 为 `foxglove.SceneUpdate`
- Optional camera publisher: `/unity/camera`，schema 为 `foxglove.CompressedImage`

在 Play Mode 中移动或旋转 Cube。观察 Foxglove 3D panel 中的 cube 更新，并确认 panel 中的位置数据同步变化。

![foxglove-live-updates](../Pictures/01/foxglove-live-updates.png)
<figcaption>图 23：Foxglove 实时更新验证</figcaption>

## 9. 预期结果

- Foxglove **Topics** panel 列出你的 Unity topics。
- **3D** panel 可以显示 Cube frame 或 primitive。
- **Image** panel 可以显示 `/unity/camera`。
- 在 Unity 中移动 Cube 会实时更新 Foxglove。
