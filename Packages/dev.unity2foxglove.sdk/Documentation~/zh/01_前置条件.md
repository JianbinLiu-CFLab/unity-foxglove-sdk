## 1. 目的

本页用于准备第一次使用 Unity2Foxglove 所需的软件。

本页刻意保持简短，不解释 package 内部依赖、sample 实现细节或网络故障排查。这些内容会在后续手册页面中说明。

## 2. 必需软件

继续快速上手前，请先安装这些软件。

| 软件 | 网站 | 要求 | 说明 |
|---|---|---|---|
| Unity Editor | [Unity 官方网站](https://unity.com/) | Unity 6000.0 LTSC 或更新版本 | SDK 核心兼容 6000.0.74f1 LTSC 或更新版本。仓库 demo project 基于 6000.3.14f1 LTSC 开发和测试。不支持 Unity 2022。 |
| Foxglove Desktop | [Foxglove 下载页](https://foxglove.dev/download) | 最新稳定版 desktop app | 用于连接 Unity、查看 topic、3D 数据、图像、参数、服务和 MCAP 文件。 |
| Git | [Git 下载页](https://git-scm.com/downloads) | 任意较新版本 | 如果你要 clone 仓库，或通过 Git URL 安装 package，则需要 Git。 |

使用 Unity2Foxglove 不需要 ROS。

> [!NOTE]
> 目前人工验证主要在 Windows 10 LTSC 平台完成。其他 desktop 平台尚未完整验证。如果遇到兼容性问题，可以联系维护者、在 GitHub 提 issue，或提交包含平台信息和复现步骤的 pull request。

## 3. 推荐 Unity 安装方式

建议通过 Unity Hub 安装 Unity。安装 Unity Editor 时，如果 Unity Hub 提供 Visual Studio 相关选项，建议保持启用。对大多数 Windows 用户来说，这是最简单的 C# 编辑和 Unity 集成方式。

只在 Unity Editor 中按 Play 运行时，标准 Unity 安装即可。

如果要构建 standalone Player，还需要安装目标平台对应的 Unity module：

| 目标平台 | Unity Hub module |
|---|---|
| Windows | Windows Build Support with IL2CPP |
| Linux | Linux Build Support with IL2CPP |
| macOS | macOS Build Support |

如果你现在只想在 Unity Editor 中跑 quick start，可以暂时跳过 Player build modules。

## 4. 可选开发工具

这些工具用于写代码、运行验证测试或执行仓库脚本。仅仅导入 package 并按 Play 不需要全部安装。

| 工具 | 网站 | 使用场景 |
|---|---|---|
| Visual Studio | [Visual Studio 下载页](https://visualstudio.microsoft.com/downloads/) | 编写 C# 脚本、自定义 publisher、`[FoxRun]` 字段、Parameters 或 Services。Windows 上推荐默认使用。 |
| JetBrains Rider | [Rider 产品页](https://www.jetbrains.com/rider/) | 偏好 Rider 的 Unity/C# 开发体验时使用。 |
| Visual Studio Code | [VS Code 下载页](https://code.visualstudio.com/) | 需要轻量编辑器，并配合 C# / Unity 扩展使用。 |
| Python 3 | [Python 下载页](https://www.python.org/downloads/) | 运行仓库辅助脚本，例如 `Scripts/build_tools/unity_il2cpp.py`。 |
| .NET SDK | [.NET 下载页](https://dotnet.microsoft.com/download) | 从命令行运行 runtime validation tests 或 performance baselines。 |

## 5. 首次运行检查清单

继续前确认：

- Unity 可以打开你的项目，并且没有 package error。
- Foxglove Desktop 已安装。
- 你可以在 Unity 中按 Play。
- 如果计划构建 standalone Player，已经安装对应 Unity build module。
- 如果计划运行仓库脚本，终端中可以使用 Python。

最小首次测试请继续阅读 [02_安装与快速上手](02_安装与快速上手.md)。

如果你想直接使用 package sample，而不是从空场景搭建，请使用 [05_基础可视化](05_基础可视化.md)。
