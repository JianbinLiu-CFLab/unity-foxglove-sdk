# 1. 常见问题排查

## 1.1 目的

这份文档用于排查连接、topic、schema、构建、FoxRun、MCAP 等常见问题。

## 1.2 应用场景

当 Foxglove 连不上、连接后没有数据、Player 和 Editor 行为不一致，或 MCAP 回放结果不符合预期时，先从这里开始。

## 1.3 Foxglove 无法连接

### 检查端口

确认 `FoxgloveManager` 的 **Port** 字段是 `8765`，Foxglove 中连接的 URL 是 `ws://127.0.0.1:8765`。

### 检查防火墙

Windows 防火墙可能阻止端口访问。Unity 首次启动 WebSocket 服务器时，系统会弹出防火墙提示，点击"允许访问"。

如未弹出，手动添加：
1. 控制面板 > Windows Defender 防火墙 > 允许应用通过防火墙
2. 确保 Unity Editor 或构建的 Player .exe 在允许列表中

### 0.0.0.0 vs 127.0.0.1

- `127.0.0.1` 仅监听本机连接（默认设置）
- 如需从其他设备连接，将 `Host` 设置为 `0.0.0.0`，Foxglove 连接该机器的局域网 IP

修改 FoxgloveManager 的 `_host` 字段（Inspector 中 Host），然后 Foxglove 中 URL 改为 `ws://<IP地址>:8765`。

### 确认 Unity 在 Play Mode

FoxgloveManager 仅在 Play Mode 且 `_startOnEnable = true` 时启动服务器。Console 中应有：
```
[Foxglove] Server started on ws://127.0.0.1:8765
```

如无此日志，检查 `FoxgloveManager` 是否挂到了场景中的 GameObject，以及 `Start On Enable` 是否勾选。

## 看不到话题

### Topics 面板为空

可能原因：
1. **未进入 Play Mode** -- FoxgloveManager 仅在 Play 时启动
2. **连接在 Play 之前** -- 如果 Foxglove 在 Play 之前已连接，会话尚未启动。断开并重新连接
3. **FoxgloveManager.Start On Enable 未勾选** -- 需手动勾选或通过代码调用 `StartServer()`

解决方法：确认进入 Play Mode 后，在 Foxglove 中断开并重新连接。

### 特定话题不显示

- `/tf`：确认场景中有 GameObject 挂载了 `FoxgloveTransformPublisher`
- `/scene`：确认有 `FoxgloveSceneCubePublisher`
- `/unity/camera`：确认 Camera 上有 `FoxgloveCameraPublisher`
- `/debug/*`：确认有挂载带 `[FoxRun]` 的 partial 类脚本

### Publisher 未找到 Manager

Publisher 组件在 `OnEnable` 时自动查找场景中的 `FoxgloveManager`。如提示 "No FoxgloveManager found in scene"，确保：
1. FoxgloveManager 的 GameObject 在场景中处于激活状态
2. Publisher 和 Manager 在同一场景中

## Schema not found 错误

当 `PublishJson` 使用了未注册的 schema 名称时出现。

### 常见原因

1. **Schema 名称拼写错误** -- 确认大小写，如 `foxglove.FrameTransform`（驼峰命名）
2. **使用了不存在的 schema** -- 仅内置 schema 可用，参考 `DefaultSchemaRegistry` 注册列表
3. **[FoxRun] SchemaName 填写错误** -- 检查 SchemaName 参数是否使用了正确的 schema 名称

### 正确用法

```csharp
// schemaless（无需注册 schema）-- 始终可用
[FoxRun("/debug/test")]
private float _val;

// schema 发布 -- 3D/Plot 面板可渲染
[FoxRun("/debug/tf", SchemaName = "foxglove.FrameTransform")]
private Vector3 _pos;
```

## [FoxRun] 话题不发布

### 编译报错 FOXRUN001

类未声明为 `partial`。添加 `partial` 关键字：

```csharp
// 错误
public class MyLogger : MonoBehaviour { /* ... */ }

// 正确
public partial class MyLogger : MonoBehaviour { /* ... */ }
```

### Play Mode 无 topic

1. 脚本未挂到 GameObject -- 将脚本拖到场景中的 GameObject 上
2. Hub 扫描间隔为 2 秒 -- 等待 3-5 秒后再查看
3. 脚本所属 GameObject 未激活 -- 确保 `isActiveAndEnabled` 为 true

### FOXRUN002 / FOXRUN003 警告

- **FOXRUN002**：同一 topic 的多个字段指定了不同的 `SchemaName`，可能导致未预期的行为。统一 SchemaName 或拆分为不同 topic。
- **FOXRUN003**：去除下划线后字段名碰撞（如 `_pos` 和 `pos` 在发布 JSON 中都会变成 `pos`）。修改字段名避免碰撞。

## IL2CPP 构建失败

### MissingMethodException / TypeLoadException

IL2CPP 代码剥离移除了反射使用的类型。检查：

1. **link.xml 是否存在** -- 确认 `Runtime/link.xml` 内容正确且未被意外覆盖
2. **Newtonsoft.Json 是否被保护** -- `<assembly fullname="Newtonsoft.Json" preserve="all" />`
3. **Unity.FoxgloveSDK 是否被保护** -- `<assembly fullname="Unity.FoxgloveSDK" preserve="all" />`

### 构建日志无 [FoxrunBuildPreprocess]

`FoxrunBuildPreprocess.OnPreprocessBuild` 应在构建早期阶段执行。如无日志输出：

1. 确认 `FoxrunBuildPreprocess.cs` 位于 `Editor/` 目录
2. 确认 `callbackOrder = -100`（负数确保在其他处理器之前）
3. 检查 Editor 程序集是否包含 `FoxrunBuildPreprocess` 和 `FoxrunCodeGenerator`

### 压缩 DLL 平台错误

`Runtime/Plugins/compression/` 下的 DLL 需要正确平台设置：

1. 在 Unity 中选中 `IonKiwi.lz4.dll` 和 `ZstdSharp.dll`
2. Inspector 中确认 **Exclude Platforms** 仅包含 **WebGL**
3. 确认 `.asmdef` 中平台排除配置正确

## MCAP 文件损坏

### 无法打开 .mcap 文件

MCAP 文件必须在 `Close()` 正确调用后才能读取。`Close()` 负责写入：
- Footer 记录（包含 Summary 区的偏移和 CRC）
- 结尾 Magic 字节（文件完整性标记）
- Flush 数据到磁盘

可能原因：
1. **Unity 异常退出** -- 在 Play Mode 期间 Unity 崩溃或强制关闭
2. **直接 kill 进程** -- 未等待 onDestroy 调用
3. **磁盘空间不足** -- 文件写入不完整

### 解决方法

- 始终通过停止 Play Mode 正常结束录制
- 确保磁盘有足够的可用空间（至少几百 MB）
- 不要手动删除或移动录制目录中的 .tmp 文件

## Parameters 不可写

### 原因

参数注册时 `Writable` 标志为 `false`，或通过 `FoxgloveParameterComponent` 注册时 `Writable` 未勾选。

### 解决方法

```csharp
// 代码注册
manager.RegisterParameter("/my_param", value, "number", writable: true);
```

或在 Inspector 中 `FoxgloveParameterComponent` 的 `Writable` 勾选。
`

### Parameters 面板无参数

1. 确认 `FoxgloveParameterComponent` 挂在与 `FoxgloveManager` 同个 GameObject
2. 确认参数列表中的 `Name` 不为空
3. 确认 FoxgloveManager 已启动

## Service 调用超时

### 超时机制

`FoxgloveServiceRegistry.DefaultTimeout` 为 10 秒。`SweepTimeouts` 定期检查超时的服务调用并自动标记失败。

### 原因

1. 服务未注册 handler delegate
2. 服务处理逻辑耗时过长（如涉及 IO 或等待）
3. 未在主线程调用 `DrainServiceCalls()`

### 解决方法

```csharp
// 注册带 handler 的服务
manager.Runtime.RegisterService(descriptor, request =>
{
    // handler 在主线程执行
    return JToken.FromObject(new { status = "ok" });
});
```

## 端口已被占用

### 现象

启动时 Console 无 `Server started` 日志，或出现端口绑定错误。

### 解决方法

1. **停止之前的 Unity 实例** -- 每个 Unity 进程只能绑定一个 8765 端口
2. **检查端口占用** -- 在终端运行 `netstat -ano | findstr 8765`（Windows）或 `lsof -i :8765`（Linux/macOS），查看是否有其他进程占用
3. **修改端口** -- 更改 FoxgloveManager 的 `Port` 为非冲突端口
