# 1. IL2CPP 构建指南

IL2CPP（Intermediate Language To C++）是 Unity 独立平台构建的标准后端。本指南说明将 Unity2Foxglove SDK 构建为 IL2CPP Player 的要点。

## 1.1 目的

这份文档用于说明如何在 IL2CPP Player 中构建和验证 Unity2Foxglove。

## 1.2 应用场景

当你准备发布 Player、验证 `[FoxRun]` 生成代码 fallback、检查 `link.xml` 或排查 Editor 与 Player 行为差异时，阅读这份文档。

## 1.3 为什么需要 IL2CPP

- Unity 独立平台构建（Windows/Linux/macOS）默认使用 IL2CPP
- IL2CPP 在构建时将 C# IL 转换为 C++ 并编译为原生代码
- 与 Mono 后端不同，IL2CPP 会进行代码剥离（code stripping），可能移除反射使用的类型
- IL2CPP 不支持运行时 Roslyn ISG，需要 fallback 方案生成源代码

## link.xml 保护

IL2CPP 代码剥离可能移除 SDK 依赖的类型，需要在项目中保留 `link.xml`。

SDK 已在包内包含 `Runtime/link.xml`：

```xml
<linker>
  <!-- Preserve Newtonsoft.Json -- heavy reflection usage -->
  <assembly fullname="Newtonsoft.Json" preserve="all" />
  <!-- Preserve Foxglove SDK runtime -->
  <assembly fullname="Unity.FoxgloveSDK" preserve="all" />
</linker>
```

Unity 在构建 IL2CPP 时会自动收集所有 Assemblies 下的 `link.xml`。如项目中另有 link.xml，确保不与之冲突。

## [FoxRun] 源代码生成

### 问题

`[FoxRun]` 依赖 Roslyn Incremental Source Generator（ISG）在 Editor 编译时生成 `_FoxRun.g.cs`。但 IL2CPP 构建时：
- Roslyn ISG 的 DLL 不会在 IL2CPP 编译管道中运行
- 因此 `IFoxgloveLogSource` 的实现代码不存在

### 解决方案：两阶段生成

#### 阶段 1：Editor 模式（ISG）

在 Unity Editor 中，`FoxgloveLogSourceGenerator`（Roslyn ISG）自动运行，生成代码存在于编译内存中，无物理文件。

DLL 位置：`Editor/SourceGenerators/analyzers/dotnet/cs/FoxgloveLogSourceGenerator.dll`

#### 阶段 2：构建前 fallback（IPreprocessBuildWithReport）

在 IL2CPP 构建开始前，`FoxrunBuildPreprocess` 自动执行：

1. 扫描所有程序集中带 `[FoxRun]` 的 partial 类
2. 调用 `FoxrunCodeGenerator.GenerateSourceFiles()` 生成真实 `.g.cs` 文件
3. 文件输出到 `Assets/Scripts/Generated/` 目录
4. 调用 `AssetDatabase.Refresh(ForceSynchronousImport)` 使 Unity 编译新文件
5. 生成的代码包裹在 `#if !UNITY_EDITOR` 中，仅在 Player 编译时参与

Console 日志示例：

```
[FoxrunBuildPreprocess] Generating FoxRun source files...
[FoxrunCodeGenerator] Generated TestLog_FoxRun.g.cs
[FoxrunBuildPreprocess] Generated 1 file(s): TestLog_FoxRun.g.cs
```

### 生成文件格式

```csharp
// <auto-generated/>
// Generated as a physical fallback for Player/IL2CPP builds.
#if !UNITY_EDITOR
#pragma warning disable
using UnityEngine;
using UnityEngine.Scripting;
using Unity.FoxgloveSDK.Components;

partial class TestLog : IFoxgloveLogSource
{
    int IFoxgloveLogSource.FoxgloveLog_TopicCount => 2;

    FoxgloveLogTopicInfo IFoxgloveLogSource.FoxgloveLog_GetTopic(int index) { /* ... */ }

    [Preserve]
    void IFoxgloveLogSource.FoxgloveLog_Publish(int topicIndex, FoxgloveManager mgr, ulong nowNs) { /* ... */ }
}
#endif
```

### 生成文件管理

- 文件位于 `Assets/Scripts/Generated/`，应在 `.gitignore` 中忽略
- 每次构建前自动重新生成（仅当内容变化时才写入，防止无意义重编译）
- 构建完成后可手动清理

## 构建脚本

项目提供 Python 构建脚本 `Scripts/build_tools/unity_il2cpp.py`：

```bash
# 构建 Windows x64 IL2CPP Player
python Scripts/build_tools/unity_il2cpp.py --target win64

# 构建 Linux x64 IL2CPP Player
python Scripts/build_tools/unity_il2cpp.py --target linux64

# 构建 macOS IL2CPP Player
python Scripts/build_tools/unity_il2cpp.py --target macos --unity /path/to/Unity.app
```

### 构建输出

构建文件输出到：`build/Unity/<target>-il2cpp-<timestamp>/`

示例：`build/Unity/win64-il2cpp-20260505_143021/`

### 构建日志监控

脚本会实时监控 Unity 构建日志，高亮输出以下标记：
- `[Foxrun` -- FoxRun 生成相关日志
- `Build succeeded` / `Build failed` -- 构建结果
- `error CS` -- C# 编译错误
- `Exception` -- 异常
- `IL2CPP` -- IL2CPP 转换进度

## 验证 IL2CPP Player

构建完成后验证：

1. 运行构建的 `.exe`（Windows）或对应平台的可执行文件
2. Player 启动后 FoxgloveManager 自动启动 WebSocket 服务器
3. 打开 Foxglove Desktop 连接 `ws://127.0.0.1:8765`
4. 确认 Topics 面板出现所有预期话题
5. 确认 3D 面板正常显示场景实体
6. 确认 `[FoxRun]` 标注的 `/debug/*` 话题正常发布

## 压缩 DLL 平台设置

`Runtime/Plugins/compression/` 下的压缩 DLL 需要正确的平台设置：

### IonKiwi.lz4.dll (.meta 设置)

- **Include Platforms**: 所有平台
- **Exclude Platforms**: WebGL

### ZstdSharp.dll (.meta 设置)

- **Include Platforms**: 所有平台
- **Exclude Platforms**: WebGL

SDK 的 `.asmdef` 文件已配置相应的平台排除。

## 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| IL2CPP 构建报 `MissingMethodException` | Newtonsoft.Json 被剥离 | 确保 `link.xml` 中 `Newtonsoft.Json` 标记为 `preserve="all"` |
| IL2CPP 构建报类型缺失 | `Unity.FoxgloveSDK` 程序集被剥离 | 确保 `link.xml` 中 `Unity.FoxgloveSDK` 标记为 `preserve="all"` |
| Player 中 `[FoxRun]` 话题不发布 | .g.cs 未生成 | 检查构建日志是否有 `[FoxrunBuildPreprocess]` 输出 |
| 压缩异常 | DLL 平台设置错误 | 检查 `.meta` 文件中的平台包含/排除设置 |
| 构建日志无 `[FoxrunBuildPreprocess]` 输出 | `FoxrunBuildPreprocess` 未注册 | 确认 `callbackOrder = -100` 在其他处理器之前 |
| `FoxrunCodeGenerator` 找不到类型 | 反射扫描失败 | 检查程序集是否可加载，跳过 `ReflectionTypeLoadException` |
