# 1. [FoxRun] -- Zero-Code Auto-Publishing

`[FoxRun]` is a C# attribute (actual class name `FoxRunAttribute`). Annotate a field or property and its value is automatically published to Foxglove at the specified rate -- no manual channel registration, serialization, or Update loop code required.

## 1.1 Purpose

Use this guide to publish debug values from Unity scripts with minimal code.

## 1.2 Application

Use `[FoxRun]` for quick observability: positions, health values, state strings, counters, and other runtime diagnostics that do not need a custom publisher component.

## 1.3 Basic usage

```csharp
using UnityEngine;
using Unity.FoxgloveSDK.Components;

public partial class TestLog : MonoBehaviour       // Must be partial
{
    [FoxRun("/debug/position")]                    // Default 10 Hz, schemaless JSON
    private Vector3 _pos;

    [FoxRun("/debug/health", RateHz = 5)]           // Custom 5 Hz rate
    private float _health = 100f;

    [FoxRun("/debug/status", SchemaName = "")]      // Empty SchemaName = schemaless JSON
    private string _status = "idle";

    void Update()
    {
        _pos = transform.position;
        _health -= Time.deltaTime;
    }
}
```

## 1.2 Requirements

| Requirement | Details |
|-------------|---------|
| Class must be `partial` | Otherwise the compiler reports error `FOXRUN001` |
| Inherits `MonoBehaviour` | Hub scans via `FindObjectsByType<MonoBehaviour>` |
| Attached to a GameObject | Must be on a scene GameObject |

## 1.3 Attribute parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `topic` | `string` | Required | Foxglove topic name, e.g., `"/debug/position"` |
| `RateHz` | `float` | `10f` | Publish frequency in Hz |
| `SchemaName` | `string` | `""` | Schema name. Empty = schemaless JSON. Fill in a registered schema name for 3D panel rendering. |

### 1.3.1 SchemaName parameter

When `SchemaName` is specified, the SDK calls `RegisterSchemaChannel` instead of a plain channel, enabling Foxglove's 3D/Plot panels to render structured data:

```csharp
[FoxRun("/debug/pose", SchemaName = "foxglove.FrameTransform")]
private Vector3 _pose;  // 3D panel recognizes and renders this
```

Supported built-in schema names (see `DefaultSchemaRegistry`):
- `foxglove.FrameTransform`
- `foxglove.SceneUpdate`
- `foxglove.CompressedImage`

## 1.4 Supported types

| C# type | Publish format |
|---------|---------------|
| `Vector3` | `{"x":1.0,"y":2.0,"z":3.0}` |
| `Vector2` | `{"x":1.0,"y":2.0}` |
| `Quaternion` | `{"x":0.0,"y":0.0,"z":0.0,"w":1.0}` |
| `Color` | `{"r":1.0,"g":0.0,"b":0.0,"a":1.0}` |
| `float`, `int`, `string`, `bool`, `enum` | Emitted as-is |

The types above are auto-expanded to JSON objects. Other types emit `.ToString()` directly.

## 1.5 Code generation mechanism

### 1.5.1 Editor mode (ISG)

During Unity Editor compilation, `FoxgloveLogSourceGenerator` (Roslyn Incremental Source Generator) generates a `_FoxRun.g.cs` file for every partial class containing `[FoxRun]`, implementing the `IFoxgloveLogSource` interface.

Generated code example (for `TestLog`):

```csharp
// <auto-generated/>
partial class TestLog : IFoxgloveLogSource
{
    int IFoxgloveLogSource.FoxgloveLog_TopicCount => 2;

    FoxgloveLogTopicInfo IFoxgloveLogSource.FoxgloveLog_GetTopic(int index)
    {
        switch (index)
        {
            case 0: return new FoxgloveLogTopicInfo("/debug/position", 10f);
            case 1: return new FoxgloveLogTopicInfo("/debug/health", 5f);
            default: return default;
        }
    }

    void IFoxgloveLogSource.FoxgloveLog_Publish(int topicIndex, FoxgloveManager mgr, ulong nowNs)
    {
        switch (topicIndex)
        {
            case 0: mgr.PublishJson("/debug/position", "", new { position = new { x = _pos.x, y = _pos.y, z = _pos.z } }, nowNs); break;
            case 1: mgr.PublishJson("/debug/health", "", new { health = _health }, nowNs); break;
        }
    }
}
```

ISG DLL location: `Editor/SourceGenerators/analyzers/dotnet/cs/FoxgloveLogSourceGenerator.dll`

### 1.5.2 IL2CPP build (fallback)

ISG does not run during IL2CPP compilation. Therefore, before building a Player, `FoxrunBuildPreprocess` (`IPreprocessBuildWithReport`) calls `FoxrunCodeGenerator.GenerateSourceFiles()` to generate real `.g.cs` files into `Assets/Scripts/Generated/`.

The generated files are named `{ClassName}_FoxRun.g.cs`, wrapped in `#if !UNITY_EDITOR`, and are only compiled during Player builds.

### 1.5.3 Generated file comparison

| Mode | Generation method | File location |
|------|-------------------|---------------|
| Editor | ISG (Roslyn in-memory) | No physical file |
| IL2CPP Build | IPreprocessBuildWithReport | `Assets/Scripts/Generated/` |

## 1.6 FoxgloveLogHub

`FoxgloveLogHub` is an auto-created singleton GameObject (`[FoxRunHub]`) that drives the rate scheduling for all `[FoxRun]`:

- Auto-created via `[RuntimeInitializeOnLoadMethod]` after scene load
- Survives scene destruction (`DontDestroyOnLoad`)
- Scans all `IFoxgloveLogSource` in the scene every **2 seconds**, auto-discovering newly attached scripts
- Per-topic independent timers, triggering publish at each topic's configured `RateHz`
- Automatically cleans up references to destroyed MonoBehaviours

## 1.7 Cooldown behavior

- Each topic has its own cooldown: `cooldown = 1f / RateHz`
- A publish is triggered only when the cooldown timer reaches zero, then it resets
- `RateHz = 0` disables limiting (publishes every frame -- use cautiously)
- When multiple fields share the same topic, the highest `RateHz` is used

## 1.8 Diagnostics

The ISG reports the following diagnostics:

| Diagnostic ID | Severity | Trigger condition |
|---------------|----------|-------------------|
| `FOXRUN001` | Error | Class is not declared `partial`. Add the `partial` keyword. |
| `FOXRUN002` | Warning | Multiple fields on the same topic specify different `SchemaName` values. |
| `FOXRUN003` | Warning | Field name collision after underscore removal (e.g., `_pos` and `pos`). |

## 1.9 Common issues

| Symptom | Cause | Resolution |
|---------|-------|------------|
| Compile error: Class not partial | FOXRUN001 | Add the `partial` keyword to the class |
| No topic in Play Mode | Script not attached to GameObject | Drag the script onto a scene GameObject |
| No topic in Play Mode | Hub hasn't scanned yet | Wait 2-4 seconds; the Hub scans every 2 seconds |
| No topic in non-Editor | IL2CPP .g.cs not generated | Check `FoxrunBuildPreprocess` build log |
| Values not updating | Update() not assigning | Ensure field values are updated in `Update`/`FixedUpdate` |
| Schema not found | SchemaName typo | Check schema name against `DefaultSchemaRegistry` |
