## Fixes Applied

- Bounded replay callback delivery with a frame and byte cap, throttled overflow warnings, and test hooks that now exercise the same enqueue/drain path as production.
- Split Unity manager client events into separate bounded queues for lifecycle events and message events so connect/disconnect does not compete with large client payloads.
- Hardened `BoundedEventQueue` accounting by measuring item size at enqueue time, treating `maxBytes <= 0` as no byte cap, and exposing cumulative dropped event/byte counters.
- Documented atomic batch semantics for subscribe and client advertise in the architecture protocol constraints.
- Made performance thresholds mode-aware so quick and full runs can use different baselines.
- Removed static threshold state from the performance runner and passed threshold config through the scenario call chain.
- Added bad-threshold-config fallback in the performance entry point. Invalid JSON or unreadable threshold files now fall back to built-in defaults with a warning.
- Expanded performance threshold self-test to cover both fail and pass paths.
- Added Gen0/Gen1 and total allocation thresholds to the checked-in JSON.
